# Security Policy

If you believe you have found a security vulnerability in Clingfy, please do **not open a public issue**.

Security issues should be reported privately so they can be investigated and resolved before public disclosure.

## Reporting

Send security reports to:

`contact@clingfy.com`

Please include as much detail as possible:

- a description of the issue
- affected versions or commit range, if known
- reproduction steps
- impact assessment
- screenshots or proof-of-concept when relevant
- diagnostic logs if available

## Retrieving diagnostic logs

If the issue occurs while using the Clingfy application, attaching the relevant diagnostic log can help us reproduce and investigate the problem faster.

You can reveal the current log file from within the application:

`Settings → Diagnostics → Reveal Today's Log File`

This will open the folder containing the log file for the current day.  
Attach the relevant log file when reporting the issue if it contains useful information about the failure.

Please review the log file before sending it to ensure it does not contain any information you do not wish to share.

## Response expectations

- We aim to acknowledge security reports within **5 business days**.
- We may ask follow-up questions if additional information is needed to reproduce the issue.
- Please avoid publicly disclosing the vulnerability until we have had a reasonable opportunity to investigate and remediate it.
